# App Icons in Apps List Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Display each app's launcher icon (48×48px) in the parent's Apps list, falling back to an initials circle when no icon is available.

**Architecture:** Java encodes the icon as base64 PNG in both the initial sync (`getInstalledPackages`) and on new installs (`PackageMonitorModule`). The `iconBase64` field rides the existing data path alongside `appName` — no new IPC methods or Hyperbee keys. `AppRow` in `AppsTab.jsx` renders an `<img>` when the field is present, or a deterministically-coloured initials circle otherwise.

**Tech Stack:** Java (Android SDK `Bitmap`, `Canvas`, `Base64`), React (WebView UI)

## Task 1: Encode icon in `getInstalledPackages()`

**Files:**
- Modify: `android/app/src/main/java/com/pearguard/UsageStatsModule.java`

- [ ] **Step 1: Add the icon encoding block inside the `for` loop in `getInstalledPackages()`**

  Locate the block that builds each `item` (around line 231). After `item.putString("appName", appName)` and before `result.pushMap(item)`, insert:

  ```java
  try {
      android.graphics.drawable.Drawable drawable = pm.getApplicationIcon(ai.packageName);
      android.graphics.Bitmap bitmap = android.graphics.Bitmap.createBitmap(144, 144, android.graphics.Bitmap.Config.ARGB_8888);
      android.graphics.Canvas canvas = new android.graphics.Canvas(bitmap);
      drawable.setBounds(0, 0, 144, 144);
      drawable.draw(canvas);
      java.io.ByteArrayOutputStream baos = new java.io.ByteArrayOutputStream();
      bitmap.compress(android.graphics.Bitmap.CompressFormat.PNG, 100, baos);
      item.putString("iconBase64", android.util.Base64.encodeToString(baos.toByteArray(), android.util.Base64.NO_WRAP));
  } catch (Exception ignored) {}
  ```

  > **Why 144×144?** Android launcher icons are typically drawn at 48dp. On a 3× density screen that's 144px — rendering at 144 then letting the WebView scale to 40px CSS gives a crisp result on all densities without excessive byte size.

    window.callBare.mockResolvedValue(policyWithoutIcon);

    render(<AppsTab childPublicKey="abc123" />);
    // Package name 'com.android.chrome' → first word 'com' → initials 'C'

    expect(initials).toBeInTheDocument();
  });

## Task 4: Implement icon and initials in `AppRow`

**Files:**
- Modify: `src/ui/components/AppsTab.jsx`

- [ ] **Step 1: Add the `getInitials` and `getIconColor` helpers at the top of the file (before `AppRow`)**

  ```js
  const ICON_COLORS = ['#4285f4','#ea4335','#fbbc05','#34a853','#ff6d00','#46bdc6','#7b1fa2','#c62828'];

  function getInitials(name) {
    return name.split(/\s+/).filter(Boolean).slice(0, 2).map(w => w[0].toUpperCase()).join('');
  }

  function getIconColor(str) {
    let hash = 0;
    for (let i = 0; i < str.length; i++) hash = str.charCodeAt(i) + ((hash << 5) - hash);
    return ICON_COLORS[Math.abs(hash) % ICON_COLORS.length];
  }
  ```

- [ ] **Step 2: Add the icon/initials element to `AppRow`'s render**

  In `AppRow`, find this block (lines 36–42 of `AppsTab.jsx`):

  ```jsx
  <div style={styles.appInfo}>
    <div style={styles.appNameBlock}>
      <span style={styles.appName}>{appData.appName || packageName}</span>
      {appData.appName && <span style={styles.pkgName}>{packageName}</span>}
      {addedDate && <span style={styles.addedDate}>Added {addedDate}</span>}
    </div>
  </div>
  ```

  Replace it with:

  ```jsx
  <div style={styles.appInfo}>
    {appData.iconBase64 ? (
      <img
        src={`data:image/png;base64,${appData.iconBase64}`}
        alt={`${appData.appName || packageName} icon`}
        style={styles.appIcon}
      />
    ) : (
      <div
        style={{
          ...styles.initialsCircle,
          backgroundColor: getIconColor(appData.appName || packageName),
        }}
        aria-hidden="true"
      >
        {getInitials(appData.appName || packageName)}
      </div>
    )}
    <div style={styles.appNameBlock}>
      <span style={styles.appName}>{appData.appName || packageName}</span>
      {appData.appName && <span style={styles.pkgName}>{packageName}</span>}
      {addedDate && <span style={styles.addedDate}>Added {addedDate}</span>}
    </div>
  </div>
  ```

  The `actions`/`controls` divs are siblings of `appInfo` and remain unchanged.
