## What Was Done This Session (PR #2, merged)

### TODO #7 — Remove "Pending Approval" badge from Apps list
Removed the yellow badge from `AppsTab.jsx` — the Approve/Deny buttons already convey pending state.

### TODO #9 — Show app display name instead of package name
- `AppsTab.jsx`: shows `appName` as bold primary label, `packageName` in small monospace below
- `ChildRequests.jsx`: same layout for My Requests list
- `bare-dispatch.js` `time:request` case: extracts `appName` from args and stores it in `req:*` records so overlay-originated requests carry the display name

### P2P sync fix — messages delivered promptly on foreground
Root cause: Hyperswarm connection drops when Android backgrounds the app; `peerConnected` flag could become stale-true if the connection died without a clean close event.

- `bare.js`: extended `conn.on('error', ...)` to reset `peerConnected = false` and `parentPeer = null`
- `bare-dispatch.js`: added `swarm:reconnect` dispatch case that calls `swarm.flush()`
- `app/index.tsx`: added `AppState.addEventListener('change', ...)` — triggers `swarm:reconnect` on every foreground transition

### TODO #6 + #11 — Child setup wizard (Accessibility Service + Usage Stats)
New mandatory two-step wizard shown to child device users on first launch and whenever either permission is missing.

**New files:**
- `app/child-setup.tsx` — two-step wizard screen with 1.5s polling loop, regression guard, styled icon components, `gestureEnabled: false` prevents back-swipe bypass

**Modified files:**
- `android/.../UsageStatsModule.java` — new `checkChildPermissions()` ReactMethod; private helpers `hasUsageStatsPermission()` and `isAccessibilityEnabled()`
- `app/_layout.tsx` — registered `child-setup` route
- `app/setup.tsx` — child mode now navigates to `/child-setup` instead of `/`
- `app/index.tsx` — `_mode` + `_dbReady` module-level vars; AppState listener re-triggers wizard if permissions revoked on foreground

**Flow:**
```
setup.tsx → /child-setup (step 1: Accessibility) → (step 2: Usage Stats) → /
index.tsx AppState active → checkChildPermissions() → /child-setup?step=N if missing
```

### Hyperbee key schema — unchanged from previous handoff
| Key | Device | Value |
|---|---|---|
| `identity` | both | `{ publicKey, secretKey }` (hex) |
| `mode` | both | `'parent'` or `'child'` |
| `peers:{identityPublicKeyHex}` | both | `{ publicKey, displayName, noiseKey, pairedAt, lastSeen }` |
| `policy` | child | latest policy snapshot from parent |
| `policy:{childPublicKey}` | parent | policy for that child |
| `req:{ts}:{packageName}` | child | `{ id, packageName, appName, requestedAt, status }` |
| `request:{requestId}` | parent | `{ id, packageName, appName, childPublicKey, status, ... }` |
| `alert:{childPublicKey}:{detectedAt}` | parent | bypass alert entry |
| `queue:{ts}` | child | queued P2P messages (flushed on reconnect) |
