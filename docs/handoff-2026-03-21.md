## What Was Done This Session (feature/child-parent-p2p-relay, merged PR #1)

### Block overlay enforcement
- `AppBlockerModule` (Accessibility Service) shows a full-screen overlay when a blocked/pending app comes to foreground
- PIN entry via on-screen numpad; verifies against argon2id hash using `LazySodiumAndroid` `byte[]` overload (bypasses `HexMessageEncoder` bug)
- "Incorrect PIN" shows inline in the numpad dialog (not as a toast behind the overlay)
- Schedule blackout and daily usage limit checks in `getBlockReason`

### Overlay stability fixes
- **Flash loop**: `FLAG_NOT_FOCUSABLE` on the overlay window; skip `TYPE_WINDOW_STATE_CHANGED` events for PearGuard's own package
- **Back-gesture bypass**: `isSystemOverlayPackage()` guard — skip events from system packages that have no launcher intent (e.g. `com.android.systemui`); this prevents gesture-nav from dismissing the overlay while the blocked app stays open
- **Re-trigger after dismissal**: 2-second cooldown (`recentlyDismissedPackage` + `dismissedAt`) after any `dismissOverlay()`. The blocked app fires one final `TYPE_WINDOW_STATE_CHANGED` as its activity destructs (back gesture, "Send Request" tap) — without this cooldown that event re-showed the overlay over the home screen
- **"Resend Request" button**: Once a request is in-flight, the overlay button label changes to "Resend Request" (`pendingRequestPackages` set). The block is NOT lifted while waiting (previous attempt allowed bypass)

### Bypass detection
- `EnforcementService` polls every 5 seconds; detects when Accessibility Service is disabled
- Emits `bypass:detected` → bare worklet → P2P relays `bypass:alert` to parent
- Parent stores alert in Hyperbee (`alert:{childPublicKey}:{detectedAt}`) with `appDisplayName: "Accessibility Service disabled"`
- Parent receives push notification; child gets persistent "re-enable" notification

### Alerts tab (parent)
- Shows bypass alerts and time requests in one list
- Approve/Deny buttons persist status in parent's Hyperbee (`request:{requestId}`) so navigating away and back preserves state
- 7-day auto-expiry: `alerts:list` dispatch deletes entries older than 7 days
- Real-time refresh via `alert:bypass` and `time:request:received` bare events

### Notification deep-links
- `pear://pearguard/alerts?childPublicKey=xxx` intent filter in `AndroidManifest.xml`
- Module-level `Linking` listener in `index.tsx` buffers pending navigation in `_pendingAlertsNav`; flushed after `dbReady` with 600ms delay
- `Dashboard` listens for `navigate:child:alerts` bare event → opens `ChildDetail` directly to Alerts tab

### Child decision notification
- When parent approves/denies, `app:decision` P2P message arrives on child → `handleAppDecision` updates policy + scans `req:*` entries
- Emits `request:updated` (with packageName/status); fallback emit fires even if no `req:*` entry found
- `index.tsx` intercepts `request:updated` → calls `UsageStatsModule.showDecisionNotification(appName, status)`
- `UsageStatsModule.setPolicy` calls `AppBlockerModule.clearPendingRequest(pkg)` for apps whose status resolves to allowed/blocked

### App icon + notification icon
- `app_icon.png` (1024×1024) → `ic_launcher.png` + `ic_launcher_round.png` at all mipmap densities (mdpi 48px → xxxhdpi 192px)
- Adaptive icon foreground (`ic_launcher_foreground.png`) with 1/6 safe-zone padding at each density
- `notification_icon.png` → white-on-transparent `ic_notification.png` at all drawable densities (mdpi 24px → xxxhdpi 96px)
- All `setSmallIcon` references in `UsageStatsModule`, `AppBlockerModule`, `EnforcementService` updated to `R.drawable.ic_notification`

## Architecture Reminders

### Key files
| File | Role |
|---|---|
| `android/.../AppBlockerModule.java` | Accessibility Service: overlay, PIN verify, bypass detection |
| `android/.../UsageStatsModule.java` | Usage stats, notifications, setPolicy, grantOverride |
| `android/.../EnforcementService.java` | Foreground service polling Accessibility state every 5s |
| `src/bare-dispatch.js` | All bare worklet IPC dispatch cases |
| `src/bare.js` | Hyperswarm, P2P message routing, sendToPeer/sendToParent |
| `app/index.tsx` | RN shell: IPC routing, native event listeners, deep-link handling |
| `src/ui/components/AlertsTab.jsx` | Parent alerts UI (bypass + time requests) |
| `src/ui/components/ChildRequests.jsx` | Child "My Requests" UI |
| `src/ui/components/Dashboard.jsx` | Parent dashboard; handles navigate:child:alerts |

### Hyperbee key schema
| Key | Device | Value |
|---|---|---|
| `identity` | both | `{ publicKey, secretKey }` (hex) |
| `mode` | both | `'parent'` or `'child'` |
| `peers:{identityPublicKeyHex}` | both | `{ publicKey, displayName, noiseKey, pairedAt, lastSeen }` |
| `policy` | child | latest policy snapshot from parent |
| `policy:{childPublicKey}` | parent | policy for that child |
| `req:{ts}:{packageName}` | child | `{ id, packageName, requestedAt, status }` |
| `request:{requestId}` | parent | `{ id, packageName, appName, childPublicKey, status, ... }` |
| `alert:{childPublicKey}:{detectedAt}` | parent | bypass alert entry |
| `queue:{ts}` | child | queued P2P messages (flushed on reconnect) |

### P2P message types
| Type | Direction | Handler |
|---|---|---|
| `hello` | both→both | `handleHello` — identity exchange, policy push |
| `policy:update` | parent→child | `handlePolicyUpdate` |
| `app:decision` | parent→child | `handleAppDecision` — updates policy + `req:*` status |
| `time:extend` | parent→child | `handleTimeExtend` — grants timed override |
| `time:request` | child→parent | `handleIncomingTimeRequest` |
| `bypass:alert` | child→parent | inline in `handlePeerMessage` switch |
| `app:installed` | child→parent | `handleIncomingAppInstalled` |
| `apps:sync` | child→parent | `handleIncomingAppsSync` |
| `usage:report` | child→parent | inline in `handlePeerMessage` switch |
