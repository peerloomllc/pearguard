### Pairing Flow Fixes (the main work of this session)

**Bug 1 — Listeners removed on unmount.**
When a deep link arrives, Expo Router navigates to `join.tsx`, which unmounts
`index.tsx`. The `useEffect` cleanup removed both the `Linking` listener and
the `DeviceEventEmitter 'pearguardLink'` listener. By the time `join.tsx`
emitted `pearguardLink`, nobody was listening.

*Fix:* Moved both listeners to module scope in `app/index.tsx` — they now live
for the lifetime of the JS bundle and are never removed.

**Bug 2 — `dbReady` never set on remount.**
After `join.tsx` called `router.replace('/')`, `index.tsx` remounted with fresh
state (`dbReady = false`). The early-return path (`_workletStarted && _worklet`)
exited before registering the `onEvent('ready', ...)` handler, and the stale
handler from the first mount pointed at the old component's `setDbReady`. The
component stayed on the black loading screen forever.

*Fix:* Added `setDbReady(true)` directly in the early-return path.

**Bug 3 — `bare.js init()` not idempotent.**
Re-calling `init` opened a new Hypercore on the same path, risking data
corruption.

*Fix:* Added `let _initialized = false` guard. On subsequent calls, `init()`
immediately re-emits `ready` with the current identity/mode and returns.

## Architecture Reminders

```
┌──────────────────────────────────────────┐
│  React Native (Expo) shell               │  app/index.tsx
│  - Module-level: _worklet, listeners     │
│  - Component: WebView, setDbReady        │
├──────────────────────────────────────────┤
│  WebView React UI                        │  src/ui/
│  - App.jsx, ParentApp, ChildApp          │
│  - Calls bare via window.callBare(...)   │
├──────────────────────────────────────────┤
│  Bare Worklet                            │  src/bare.js + src/bare-dispatch.js
│  - Hyperswarm, Hyperbee, identity        │
│  - _initialized guard prevents re-init  │
└──────────────────────────────────────────┘
```

### Deep link flow (pear://pearguard/join?t=...)

1. Android intent → Expo Router → `app/join.tsx`
2. `join.tsx` emits `DeviceEventEmitter.emit('pearguardLink', url)` after 1.5s
3. Module-level listener in `index.tsx` calls `sendToWorklet({ method: 'acceptInvite', args: [url] })`
4. `join.tsx` calls `router.replace('/')` after 2s
5. `index.tsx` remounts → `start()` → early-return → `setDbReady(true)` → WebView renders
6. Bare worklet processes `acceptInvite` → joins Hyperswarm topic → waits for peer
